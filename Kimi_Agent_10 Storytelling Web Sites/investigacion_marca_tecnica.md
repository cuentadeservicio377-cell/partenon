### Como Cuentan la Historia

Stripe utiliza una **narrativa de confianza global y crecimiento escalable**. La historia se construye desde lo macro hasta lo micro:

#### Progresion narrativa:

| Seccion | Funcion narrativa | Tecnica |

---

### Elementos Visuales Destacados

1. **Wave Animation (Hero)**: Gradiente organico de colores (rosa, naranja, azul) que fluye detras del texto como una ola. Esta animacion es un sello distintivo de Stripe y evoca fluidez, movimiento y naturalidad en un producto financiero tradicionalmente rigido.

2. **GDP Counter**: Un contador en tiempo real que muestra el porcentaje del PIB global que corre sobre Stripe. Es un elemento de storytelling exclusivo -- solo Stripe puede mostrar este dato -- y comunica escala de manera inmediata.

3. **Demos Interactivas de Checkout**: Simulaciones funcionales de interfaces de pago en multiples idiomas (ingles, aleman, japones) que cambian en tiempo real. Muestran diferentes escenarios: cafe, e-commerce, suscripciones.

4. **Bento Box UI**: Paneles modulares que presentan informacion densa de manera organizada y estetica, inspirados en las presentaciones de producto de Apple.

5. **Dashboard de Zenflow**: Una tabla completa de cuentas conectadas con balances y volumenes que simula una plataforma real de gestion de pagos.

---

### Por Que es un Buen Ejemplo de Storytelling Tecnico

- **Narrativa de escala progresiva**: "Desde tu primera transaccion hasta tu billonesima" -- comunica que crecen contigo
- **Show, don't tell**: En lugar de describir como funciona, te muestran el checkout funcionando
- **Metricas como prueba de autoridad**: 135+ monedas, $1.9T en volumen, 99.999% uptime, 200M+ suscripciones
- **Diversidad de casos de uso**: Desde un cafe hasta una empresa Fortune 100
- **Personificacion del producto**: Las demos tienen nombres propios (Roastery, Cartsy, Showflix)

## 2. LINEAR

### Informacion Basica

| Campo | Detalle |
|-------|---------|
| **Empresa** | Linear |
| **URL** | https://linear.app |
| **Categoria** | Product management / Issue tracking |
| **Headline principal** | "The product development system for teams and agents" |
| **Sub-headline** | "Purpose-built for planning and building products. Designed for the AI era." |

---

### Como Cuentan la Historia

Linear emplea una **narrativa evolutiva y sistemica**. La historia se cuenta a traves de una metáfora biologica de "una nueva especie de herramienta de producto".

#### Progresion narrativa:

| Seccion | Funcion narrativa | Tecnica |
|---------|------------------|---------|
| **Hero** | Inmersion inmediata en el producto | La interfaz real del producto ES el hero |
| **Logos de clientes** | Prueba social cualitativa | Vercel, Cursor, OpenAI, Coinbase, Cash App |
| **"A new species"** | Posicionamiento evolutivo | Metáfora biologica: no es un incremento, es evolucion |
| **Tres pilares** | Fundamentos del producto | Built for purpose / Powered by AI agents / Designed for speed |
| **Intake** | Automatizacion de entrada | "Make product operations self-driving" -- metáfora de conduccion autonoma |
| **Plan** | Direccion estrategica | Roadmap visual con initiatives, projects, PRDs |
| **Build** | Ejecucion con AI agents | Delegacion completa de issues a agents |

#### Arco narrativo:
> **Captura** (mira el producto funcionando) -> **Contexto** (es una nueva especie de herramienta) -> **Problema/Solucion** (Intake automatiza la entrada) -> **Estrategia** (Plan define la direccion) -> **Ejecucion** (Build con agents delega tareas) -> **Velocidad** (ship faster)

#### Estructura narrativa: Intake -> Plan -> Build
Linear estructura su historia como un flujo de trabajo completo:
1. **Intake** (captura): Convierte conversaciones en issues accionables
2. **Plan** (direccion): Define roadmap e initiatives
3. **Build** (ejecucion): Agents trabajan junto al equipo

---

### Elementos Visuales Destacados

1. **Producto como Hero**: La seccion hero NO es un texto descriptivo -- es la interfaz real del producto mostrando un issue llamado "Faster app launch" con actividad, comentarios, labels, y un AI agent (Linear Opus 4.8) trabajando en el codigo. Es la tecnica "show, don't tell" llevada al extremo.

2. **Tres Diagramas Estilizados (FIG 0.2, 0.3, 0.4)**: Ilustraciones lineares en 3D isometrico que representan:
   - Built for purpose (una estructura solida/funda)
   - Powered by AI agents (bloques interconectados)
   - Designed for speed (capas superpuestas)

3. **Interfaz de Intake**: Muestra una conversacion de Slack donde usuarios reportan un bug, y el sistema de Intake de Linear lo convierte automaticamente en issues clasificados y priorizados.

4. **Roadmap Visual**: Un timeline de Gantt con initiatives como "Core Product", "APAC Expansion", "UI Refresh" que muestra la planificacion visual del producto.

5. **Dark Theme**: El tema oscuro crea una sensacion de sofisticacion tecnica y enfasis en el contenido. Tipografia: serif para headlines (elegancia editorial), sans-serif para UI.

---

### Por Que es un Buen Ejemplo de Storytelling Tecnico

- **"The product IS the story"**: En lugar de describir Linear, te muestran Linear funcionando desde el primer segundo
- **Metáfora biologica**: "A new species of product tool" -- posiciona el producto como evolucion, no como mejora incremental
- **Metáfora de conduccion autonoma**: "Make product operations self-driving" -- comunica automatizacion inteligente
- **Narrativa de flujo**: Intake -> Plan -> Build guia al usuario a traves de un workflow completo
- **Personajes**: Los "agents" (Linear Opus 4.8) son personajes con nombre y rol en la historia
- **Numeracion como libro**: "FIG 0.2", "FIG 0.3", "FIG 0.4" -- tratan la pagina como una publicacion tecnica

## 3. VERCEL

### Informacion Basica

| Campo | Detalle |
|-------|---------|
| **Empresa** | Vercel |
| **URL** | https://vercel.com |
| **Categoria** | Cloud / Hosting / Developer platform |
| **Headline principal** | "Agentic Infrastructure" |
| **Sub-headline** | "Build agents on infrastructure that thinks like them" |

---

### Como Cuentan la Historia

Vercel utiliza una **narrativa antropomorfica por capas**. La infraestructura no es solo un servicio -- es una entidad que "piensa".

#### Progresion narrativa:

| Seccion | Funcion narrativa | Tecnica |
|---------|------------------|---------|
| **Hero** | Establecer concepto central | Triangulo geometrico + "Agentic Infrastructure" |
| **Tres lineas de texto** | Definir proposito | "For coding agents / To ship apps and agents / Automated by agents" |
| **Capa 1: Agents** | Caso de uso primario | Notion: "powers millions of agent conversations daily on Vercel" |
| **Capa 2: Apps** | Escalar a aplicaciones | Zapier: "serves over 100 million monthly website visits" |
| **Capa 3: Platforms** | Plataformas enterprise | Mintlify: "powers documentation for 20,000+ companies" |
| **Features** | Capacidades tecnicas | Durable Orchestration, Sandboxed Environments, AI Model Gateway, Fluid Compute |

#### Arco narrativo:
> **Concepto** (infrastructure that thinks) -> **Capa 1** (construye agents) -> **Capa 2** (ship apps que escalan a millones) -> **Capa 3** (host plataformas enterprise) -> **Confianza** (metricas y casos)

#### Estructura por capas:
Vercel organiza su narrativa en tres capas de uso progresivo:
1. **Para agents**: Construye agents sobre infraestructura inteligente
2. **Para apps**: Aplicaciones que escalan de cero a millones
3. **Para plataformas**: Servir a cada cliente enterprise

---

### Elementos Visuales Destacados

1. **Triangulo Geometrico (Hero)**: Un triangulo negro puro en el centro de la pagina -- minimalismo extremo. El triangulo es un simbolo de estabilidad y direccion. Representa la simplicidad de Vercel: un solo elemento geometrico para una plataforma compleja.

2. **Tres Lineas de Tag**: A la derecha del hero, tres lineas en monospace definen el proposito:
   - "FOR CODING AGENTS"
   - "TO SHIP APPS AND AGENTS"
   - "AUTOMATED BY AGENTS"
   Esto crea un mantra visual que refuerza el posicionamiento.

3. **Demos Contextuales por Capa**: Cada seccion muestra un producto real usando Vercel:
   - Notion AI chat interface
   - Zapier homepage
   - Mintlify documentation platform

4. **Lista de Features en Monospace**: Cada seccion tiene features listados en fuente monospace tipo terminal (DURABLE ORCHESTRATION, SANDBOXED ENVIRONMENTS, etc.) que refuerza la estetica developer.

5. **Tipografia Geometrica**: Fuente sans-serif geometrica (similar a Geist, la fuente de Vercel) que transmite precision tecnica. El contraste entre titulos grandes y textos pequenos en monospace crea jerarquia.

---

### Por Que es un Buen Ejemplo de Storytelling Tecnico

- **Antropomorfismo**: "Infrastructure that thinks like them" -- la infraestructura tiene inteligencia propia
- **Estructura por capas**: Tres niveles de uso (agents -> apps -> platforms) muestran versatilidad sin confusion
- **Minimalismo narrativo**: Pocas palabras, mucho significado. Cada palabra cuenta.
- **Casos de uso reales**: Notion, Zapier, Mintlify demuestran escala real
- **Triada repetida**: "For coding agents / To ship apps and agents / Automated by agents" -- repeticion ritmica

- **Storytelling visual**: Los colores, ilustraciones y layout cuentan una historia de creatividad y colaboracion

- **Contraste narrativo**: "Weekend" vs "Millions" -- la tension entre estos dos extremos captura atencion
- **Promesa de transformacion**: No solo es una herramienta, es un cambio de estado (de pequeno a grande)
- **Modularidad visual**: El grid de features muestra que es una plataforma completa pero modular
- **Developer-first**: 104.8K GitHub stars, compatibilidad con todos los frameworks, terminologia tecnica
- **Open source como storytelling**: La popularidad en GitHub es parte de la narrativa
- **Simplicidad como mensaje**: El diseno limpio refuerza el mensaje de "construye rapido"

## PATRONES COMUNES ENCONTRADOS

### 1. Narrativa con Metáfora Central
Todos los sitios usan una metáfora que guia toda la experiencia:
- **Stripe**: Infraestructura fisica (puentes, autopistas)
- **Linear**: Evolucion biologica ("new species")
- **Vercel**: Inteligencia/Consciencia ("thinks like them")
