# Plan: Hermes Parthenon — Hackathon Website

## Context
- **Hackathon**: The Hermes Agent Accelerated Business Hackathon by Nous Research x NVIDIA x Stripe
- **Submission deadline**: June 30, 2026
- **Goal**: Build a 2-page website that tells the story + shows technical architecture

## Stage 1 — Research & Asset Collection
- **Skill**: `deep-research-swarm` (lightweight, focused)
- **Tasks**:
  1. Research NVIDIA NemoClaw, Nemotron 3 Ultra, Stripe Skills for Hermes, and the Hermes framework from Nous Research
  2. Research Greek mythology of Hermes and his heroes (subtle archetypal references)
  3. Study Nous Research website aesthetic for design inspiration
  4. Find relevant metrics and statistics for small business impact (hours saved, revenue improvement, etc.)
  5. Identify Google Workspace integration patterns and MCP (Model Context Protocol) architecture
- **Output**: Research brief with findings, stats, design references, technical architecture notes

## Stage 2 — Design & Content Planning
- **Skill**: Orchestrator-designed (no specific skill needed)
- **Tasks**:
  1. Define visual design system (inspired by Nous Research — dark, minimal, elegant)
  2. Map out Page 1 structure (Marketing/Story page):
     - Hero: Parthenon allegory (subtle mythological branding)
     - Vision: Hermes as the Company, heroes as agents
     - Hero profiles (6 heroes with archetypal personalities)
     - Impact counter (10 → 1M users)
     - Outreach strategy (universities, coworking, BNI, accelerators)
     - CTA to install
  3. Map out Page 2 structure (Technical/Developer page):
     - Architecture overview with diagrams
     - Each hero's technical spec (skills, MCP connections, tools)
     - G-Brain MCP connector diagram
     - Google Workspace integration
     - Workshop technical process
     - Repository structure
  4. Write all copy for both pages
- **Output**: design.md, content.md for both pages

## Stage 3 — Web Development
- **Skill**: `vibecoding-webapp-swarm`
- **Tasks**:
  1. Build Page 1: Marketing/Story page with all sections
  2. Build Page 2: Technical/Developer page (mirror structure)
  3. Implement interactive elements (animated counter, diagrams, hero cards)
  4. Responsive design
- **Output**: Complete React webapp with both pages

## Stage 4 — Deployment & Polish
- **Tasks**:
  1. Final review and QA
  2. Deploy website
  3. Generate submission materials summary
- **Output**: Live website, summary for hackathon submission
