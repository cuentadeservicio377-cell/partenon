Proceso

De la intención a la entrega.

No instalas otra app. Dices qué necesitas y Hermes convierte tu intención en misiones con evidencia.

01

Llega la intención

El dueño describe su empresa y qué necesita ordenar. Hermes escucha y pregunta.

02

Hermes activa héroes

Según la misión, despierta al Tesorero, Mensajero, Cobrador, Guardián, Estratega o Diplomático.

03

Misiones en movimiento

Cada héroe usa su perfil, herramientas y memoria. Consultan G-Brain entre sí.

04

El Partenón entrega

El resultado vive en Sheets, Docs, Calendar o Gmail. El dueño revisa, aprueba o corrige.

Caso concreto

Un día con Hermes: Cafetería Aurora.

Aurora tiene 4 empleados y vende unas 200 tazas al día. Así opera con el Partenón.

08:00

El Tesorero detecta un gasto raro.

"El gasto en leche subió 18% esta semana. ¿Ajustamos la compra o subimos el precio del capuccino?" La alerta vive en Google Sheets.

10:00

El Mensajero publica la promo.

Crea el copy para Instagram, guarda la versión final en Google Docs y deja el presupuesto visible para el Tesorero.

12:00

El Cobrador cobra y provisiona.

Genera un payment link por $4,500 MXN. También renueva el dominio anual con Stripe Link y provisiona una base de datos Neon para la app de pedidos. Cada gasto pide aprobación en la app de Link.

15:00

El Diplomático negocia un hito.

El proveedor de insumos entrega el viernes. Operaciones necesita dos días. El Diplomático coordina la fecha y el Estratega la guarda en Calendar.

17:00

El Estratega cierra el día.

Revisa tareas pendientes, agenda recordatorios y deja un resumen en Gmail para el dueño.

Esto no es un chatbot respondiendo preguntas. Es un sistema que publica misiones, las ejecuta y deja evidencia en archivos reales.

Stack del hackathon

Agentes que cobran, pagan y operan seguros.

Partenon une tres capas: Hermes como runtime de agentes, NVIDIA como sandbox y modelo acelerado, y Stripe como infraestructura de pagos programáticos.

terminal

Hermes Agent

Runtime open source de Nous Research. Skills auto-generados, memoria persistente, MCP, cron, subagentes y un gateway a 22 plataformas de mensajería.

security

NVIDIA NemoClaw

Orquestación de agentes autónomos de larga duración. OpenShell sandboxa cada héroe con políticas de filesystem, red y privacidad.

memory

Nemotron 3 Ultra

550B parámetros MoE, 55B activos por token, contexto de 1M tokens. Hasta 5× más rápido y 30% más barato en tareas de agentes.

payments

Stripe Skills

stripe-link-cli para compras, mpp-agent para pagos por API, y stripe-projects para provisionar SaaS bajo presupuestos controlados.

Partenon se inspira en proyectos ganadores de hackathons anteriores de Hermes: la competencia multi-agente de Gladiator, el flujo creativo coordinado de JumpFoundry y la memoria de personalidad de Hermes Nordic. Aquí esas lecciones se aplican a operaciones de negocio.

Héroes del Partenón

Arquetipos operativos, no más automatización sin rostro.

Tu empresa necesita héroes: perfiles con permisos, herramientas y memoria propia. Cada uno se activa solo cuando la misión le corresponde.

Memoria compartida

Sin cerebro, los héroes no se hablan.

Hermes trae memoria nativa con MEMORY.md y USER.md. G-Brain la extiende por MCP para que cada héroe lea contexto, escriba resultados y consulte lo que otros aprendieron.

  • check Perfiles: archivos .finance, .design y .security definen comportamiento.
  • check Misiones: cada tarea queda registrada con input, output y aprendizajes.
  • check Entidades: clientes, proveedores y productos se guardan como objetos reutilizables.
  • check Skills auto-generados: tras misiones complejas Hermes crea o parchea SKILL.md para la empresa.

Auto-mejora

Hermes aprende de cada misión.

Partenon no empieza de cero cada mañana. Después de cada interacción, Hermes evalúa el resultado y propone actualizaciones de memoria y skills. El Curator consolida la biblioteca cada 7 días. Todo pasa bajo aprobación humana.

sync Closed learning loop: interacción → registro → evaluación → optimización → skill consolidada.
schedule Cron: tareas programadas en lenguaje natural o expresión cron; entrega a Telegram, WhatsApp, email o Sheets.
forum Messaging gateway: una sola sesión persiste en Telegram, WhatsApp, Slack, Discord, email y 17 plataformas más.

Instalación

Empieza en minutos.

Hermes trabaja con archivos de perfil. Los héroes publican resultados en Sheets, Docs, Slides, Drive, Calendar y Gmail. Puedes correrlo localmente o dentro de NVIDIA NemoClaw con OpenShell.

  1. 1. Clona el repositorio.
  2. 2. Corre el instalador o el onboard de NemoClaw.
  3. 3. Instala las Stripe Skills oficiales.
  4. 4. Copia el Google Sheet base y define el tipo de empresa.
Ver documentación técnica arrow_outward
Terminal 
# Opción A: instalador local
curl -fsSL https://partenon.dev/install.sh | bash

# Opción B: onboard con NVIDIA NemoClaw
curl -fsSL https://www.nvidia.com/nemoclaw.sh | NEMOCLAW_AGENT=hermes bash

# Stripe Skills oficiales
hermes skills install official/payments/stripe-link-cli
hermes skills install official/payments/mpp-agent
hermes skills install official/payments/stripe-projects

El instalador configura entorno, dependencias y el comando global hermes. NemoClaw crea un sandbox con políticas de red y filesystem. Las Stripe Skills habilitan pagos programáticos bajo aprobación humana.
