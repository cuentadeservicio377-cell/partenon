Arquitectura

Una empresa hecha de perfiles.

Partenon no es un chatbot con múltiples personalidades. Es una cola de misiones donde cada perfil tiene permisos, herramientas y memoria propia.

corporate_fare

Hermes

Recibe intenciones del emprendedor, pregunta el tipo de empresa y convierte la respuesta en misiones publicadas en el Partenón.

memory

G-Brain

Memoria persistente vía MCP. Almacena perfiles, misiones pasadas, preferencias y resultados para mantener contexto entre héroes.

badge

Héroes

Agentes con habilidades específicas. Cada uno lee su archivo de perfil correspondiente antes de actuar y entrega en Google Workspace.

Flujo

De la intención a la entrega.

El emprendedor no le da instrucciones a cada héroe. Habla con Hermes. Hermes publica la misión. El héroe correcto la toma y trabaja con el emprendedor hasta entregar.

Stack

Tecnologías planas y abiertas.

No construimos una plataforma cerrada. Partenon se monta sobre herramientas que las pymes ya usan.

language

Frontend

HTML estático, Tailwind CSS vía CDN, JavaScript vanilla. Dashboard en Next.js 15 + React 19 + TypeScript.

terminal

Core

Python 3.11+, partenon-core con router, onboarding, workflows y MCP.

workspace_premium

Workspace

Google Sheets, Docs, Slides, Drive, Calendar, Gmail. Plantillas Excel con openpyxl.

payments

Pagos

Stripe API, webhooks, payment links y suscripciones.

neurology

IA / LLMs

Nous Research, OpenAI, Kimi Coding y modelos locales según perfil.

storage

Memoria

G-Brain MCP con FastMCP para contexto persistente entre héroes.

design_services

Diseño

Open Design, redes sociales, WordPress y SSH para sitios.

account_tree

Diagramas

Mermaid para arquitectura y secuencias.

Perfiles técnicos

Cada héroe, una especificación.

Aquí no hay personalidades decorativas. Cada agente tiene un framework, skills, herramientas/Pegaso, MCP y un contrato de entrada/salida.

MCP

G-Brain vía MCP.

MCP (Model Context Protocol) es el cable entre los héroes y el cerebro. Cada agente puede leer contexto, escribir resultados y consultar lo que otros héroes aprendieron.

01 / Perfiles

Archivos .finance, .design y .security definen comportamiento.

02 / Misiones

Cada tarea queda registrada con input, output y aprendizajes.

03 / Entidades

Clientes, proveedores y productos se guardan como objetos reutilizables.

Workshop

Paquete de implementación de 90 minutos.

El mismo formato sirve para universidades, organizaciones empresariales, coworkings y aceleradoras. Los participantes se llevan Hermes instalado.

Flujo del workshop

0–20'

Setup

  • check Pre-instalación con curl ... | bash.
  • check Clonar repositorio y levantar Docker Compose.
  • check Copiar Google Sheet base y conectar Workspace.
  • check Crear archivos de perfil .finance, .design, .payments, .security, .ops, .relations.
20–60'

Onboarding

  • check Definir tipo de empresa.
  • check Configurar perfiles prioritarios.
  • check Ejecutar primera misión.
  • check Conectar Stripe y redes sociales.
60–90'

Práctica

  • check Publicar 3 misiones reales.
  • check Revisar entregables en kanban y Sheets/Docs.
  • check Ejecutar demo del Tesorero localmente.
  • check Ajustar prompts y permisos.
  • check Dejar plan de 30 días.
Instalación

Empieza en minutos.

El instalador configura entorno Python, dependencias, el comando global hermes y los seis perfiles. Luego copias el Google Sheet base y conectas las variables de entorno.

  1. 1. Clona el repositorio.
  2. 2. Corre el instalador.
  3. 3. Copia el Google Sheet base.
  4. 4. Configura variables de entorno.
  5. 5. Define el tipo de empresa.

Roadmap

Lo que viene.

Partenon se construye en público. Cada versión amplía el número de héroes activos y la profundidad de sus integraciones.

Q2 2026
v0.1.0 — Seis héroes + dashboard

Tesorero, Mensajero, Cobrador, Guardián, Estratega y Diplomático con kanban y G-Brain MCP.

Q3 2026
v0.2 — Integraciones live

Google Workspace API, Stripe webhooks, modelos locales con Nvidia y despliegue en Railway.

Q4 2026
v0.3 — Memoria avanzada

G-Brain persistente, entidades enlazadas, aprendizajes por misión y evaluación automática.

2027
v1.0 — Marketplace de misiones

Templates por industria, comunidad de implementadores y métricas de impacto.
